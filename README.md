以下是针对《136. 只出现一次的数字》这道题的详细拆解：

### （1）用白话文解释题目意思及生僻概念

**题目白话文解释：**
有一个装满数字的列表，在这个列表里，数字都是“成双成对”出现的（每个数字必然出现两次）。但是，唯独有一个数字是“单身狗”，只出现了一次。现在的任务就是要在茫茫数字中，把这个唯一落单的数字找出来。

**生僻概念解释：**
* **线性时间复杂度 ($O(N)$)**：这是对算法运行速度的一种严苛要求。意思是，寻找过程耗费的时间只能跟列表中数字的数量成正比。通俗来说，就是只能把整个列表从头到尾扫一遍（或者有限的几遍），绝对不能拿着第一个数字去跟后面所有数字挨个比对，那样太慢了。
* **常量额外空间 ($O(1)$)**：这是对算法占用内存的严苛要求。意思是，在寻找的过程中，不能申请一个新的大列表或者大本子来专门记录每个数字出现了几次。不管原列表有一百个数字还是哪怕一万个数字，手头能用来记东西的“草稿纸”只能固定那么一丁点大（比如只能申请一两个简单的变量）。
* **异或运算 (XOR, 符号为 `^`)**：一种基于二进制数字的运算。它的核心法则是：**相同为 0，不同为 1**。可以把它想象成一个“找茬”游戏，两个数对比，长得一样的部分被消除（变成0），不一样的部分被保留（变成1）。

### （2）解题思路是怎么来的（基于代码反推）

这段代码采用了非常巧妙的**位运算（异或）**思路。

1.  **排除常规思路**：
    * 想要找出现一次的数字，最符合直觉的方法是建一个“哈希表”来统计次数。但这需要消耗与数组大小成正比的内存，违背了“常量额外空间”的限制。
    * 另一种直觉是先给数组排个序，然后前后对比。但最快的排序算法也达不到“线性时间复杂度”的要求。
2.  **引入异或魔法**：为了同时满足上述两个变态级别的限制，必须跳出常规的数据结构，利用底层数字的位运算特性。异或运算有三个堪称为这道题量身定制的数学性质：
    * **性质一**：任何数和 `0` 做异或，结果还是那个数本身（`a ^ 0 = a`）。
    * **性质二**：任何数和它自己做异或，结果必定是 `0`（`a ^ a = 0`）。这就像是正反物质相撞，直接湮灭了。
    * **性质三**：异或运算满足交换律和结合律。也就是说，一堆数字连着进行异或运算，先算谁、后算谁、数字怎么打乱位置，最终的结果都是一样的（`a ^ b ^ a = a ^ a ^ b = 0 ^ b = b`）。
3.  **大一统消除**：基于以上性质，只需要把数组里所有的数字全部拿来，排着队进行异或运算。因为除了一个数字外，其余数字全都出现了两次。在不断的异或过程中，那些“成双成对”的数字无论隔得有多远，最终都会两两相撞、灰飞烟灭（变成 0）。
4.  **水落石出**：所有的对子都变成了 0，最后剩下的就是 `0 ^ 落单的数字`。根据性质一，结果就只剩下那个唯一落单的数字了。

### （3）代码框内的详细注释

```python
class Solution:
    # 注意：原截图中的返回类型 -> List[int] 属于原作者笔误，按照逻辑应返回单个整数 int
    def singleNumber(self, nums: List[int]) -> int:
        x = 0              # 初始化一个变量 x 为 0，作为异或运算的初始基底
        for num in nums:   # 依次遍历数组中的每一个数字
            x ^= num       # 将 x 与当前数字进行异或运算，成对的相同数字会在此过程中互相抵消为 0
        return x           # 循环结束后，所有对子全部归零，x 中保留的确切数值就是那个单数
```

### （4）每一行代码的详细中文解释

* `class Solution:`
    * 定义封装解题算法的类。
* `def singleNumber(self, nums: List[int]) -> int:`
    * 定义执行函数 `singleNumber`，接收一个整数列表 `nums` 作为输入。*(注：原文代码截图中标注的返回值类型 `List[int]` 是错误的，由于最终只返回一个落单的数字，正确的类型标注应为 `int`。)*
* `x = 0`
    * 设定一个用于累积计算结果的变量 `x`，初始值赋为 `0`。由于 `0` 与任何数字异或都不会改变那个数字的值，因此它是完美的起始值。这就满足了“常量额外空间”的要求，仅用到一个变量。
* `for num in nums:`
    * 开启一个循环，按顺序遍历 `nums` 列表，将每次取出的数字放入临时变量 `num` 中。这就满足了“线性时间复杂度”的要求，仅扫描数组一遍。
* `x ^= num`
    * 这是 `x = x ^ num` 的简写形式。这是整段代码的核心。随着循环的推进，`x` 不断与新的 `num` 发生异或反应。遇到第一次出现的数字，其特征会被记录在 `x` 中；当该数字第二次出现时，由于“同值异或必为0”的特性，该数字在 `x` 中留下的特征会被完美抹除。
* `return x`
    * 当遍历完所有的元素后，所有出现了两次的数字都已自我抵消。此时变量 `x` 中记录的数据，就是那个从头到尾只出现过一次的数字，将其作为结果返回。

### （5）具体数值算例及步骤表格展示

**算例设定：**
设定输入数组：`nums = [2, 1, 4, 2, 1]`。
为了直观展现异或的“消除”过程，下面表格将辅助展示对应的二进制状态（假定用4位二进制表示）：
* `2` ➔ `0010`
* `1` ➔ `0001`
* `4` ➔ `0100`

**推演过程表格：**

| 步骤 | 取出当前 `num` (二进制) | 异或运算逻辑 `x ^ num` | 更新后的 `x` (二进制) | 更新后的 `x` (十进制) | 对应代码行 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **初始** | 无 | / | `0000` | **`0`** | `x = 0` |
| **循环 1** | `2` (`0010`) | `0000 ^ 0010` | `0010` | **`2`** | `x ^= num` |
| **循环 2** | `1` (`0001`) | `0010 ^ 0001` | `0011` | **`3`** | `x ^= num` |
| **循环 3** | `4` (`0100`) | `0011 ^ 0100` | `0111` | **`7`** | `x ^= num` |
| **循环 4** | `2` (`0010`) | `0111 ^ 0010` *(这里抵消了之前的2)* | `0101` | **`5`** | `x ^= num` |
| **循环 5** | `1` (`0001`) | `0101 ^ 0001` *(这里抵消了之前的1)* | `0100` | **`4`** | `x ^= num` |
| **结束** | 遍历完毕 | / | / | 返回 **`4`** | `return x` |
